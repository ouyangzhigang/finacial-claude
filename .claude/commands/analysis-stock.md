将该文件视为本次执行的指令框架,严格按其「角色设定 / 分析框架 / 执行流程 / 输出要求」完成一次个股深度分析。本次调用额外强制:先用同花顺(iFind)采集下列行情与事件数据,再喂入框架展开分析。

## 输入变量绑定(以此覆盖框架「输入变量」的【待填】占位)

> 原理:`$ARGUMENTS` 只在本命令文件内替换,不会钻进 `@` 引用的框架文件。故在此显式绑定,模型读框架 + 本节绑定值后执行。

**原始入参**:`$ARGUMENTS`

按下表解析原始入参并填入各变量,缺失项用默认值,在报告中标注"默认/查询所得":

| 框架变量 | 解析关键词 | 缺失默认 |
|---|---|---|
| 股票名称/代码 | 首个代码(6 位)或中文名 | 无默认,必填——缺失则向用户追问,不臆测 |
| 当前价格 | "现价/当前价 X" | 实时查询最新价(注明盘中/昨收/延迟) |
| 持仓情况 | "持仓 X 股 成本 Y" | 无仓 |
| 投资周期 | "短线/波段/中长线" | 波段 |
| 风险偏好 | "保守/稳健/激进" | 稳健 |
| 账户总资产 | "账户 Xw" 或 "X 万" | 1w |

> 原始入参为空、或股票无法识别 → 先向用户追问,不臆测。
> 解析后,在报告"决策仪表盘"上方或正文起首复述一遍绑定结果,便于核对。

## 第一步:同花顺(iFind)数据采集(必做,逐项注明数据时效与来源)

| 数据维度 | iFind 工具 | 查询口径 |
|---|---|---|
| 金融热点事件 | `ifind_search_trending_news` | keyword=股票名称/所属题材,time_scope=24小时或一周,size=10 |
| 政策事件 | `ifind_search_news` | query="所属行业 + 政策/补贴/采购/监管",近 1 月,size=10 |
| 热门板块数据 | `ifind_sector_data` | 该股所属板块成分股个数、近 5/20 日平均涨跌幅、资金流向 |
| 个股行情+技术指标 | `ifind_get_stock_summary` | 近 1 月每日 OHLCV+涨跌幅+换手率+技术面 |
| 个股财务与估值 | `ifind_get_stock_financials` | ROE/净利增速/资产负债率/PE 历史分位等 |
| 上市公司事件 | `ifind_get_stock_events` | 近期 IPO/增发/重组/分红/解禁等 |
| 龙虎榜数据 | `ifind_get_stock_info`(若未返回则降级) | 近期机构/游资席位进出与接力 |
| 主力与游资流向 | `ifind_get_stock_info`(若未返回则降级) | 近 5/20 日主力净流入、超大单/大单占比 |

**降级兜底(soft-fail,不中断整体分析)**:iFind 为自然语言取数,若未直接返回龙虎榜/主力资金字段,改用 wind `wind_get_stock_technicals`(显式查"龙虎榜 + 融资融券 + 主力资金流向",需 `WIND_SSL_NO_VERIFY=1`);再不行用 akshare 历史数据 + 东方财富 push2 镜像估算。某维度最终仍缺失则标注"数据缺失",不编造。

## 第二步:按框架展开分析(数据→结论,层层咬合)

将第一步采集的数据喂入 stock-analysis-prompt-cn.md 的六大模块,数据与结论一一对应:

- 一、宏观与政策环境 ← 政策事件 + 热点事件 + 板块数据
- 二、基本面诊断 ← iFind 财务与估值 + 公司事件
- 三、技术面与量价流动性 ← iFind summary 行情 + 技术指标
- 四、资金与筹码面 ← 龙虎榜 + 主力/游资流向 + 北向 + 股东户数
- 五、情绪与事件面 ← 热点事件 + 题材热度 + 事件日历
- 六、综合结论与操作策略 ← 全部咬合后输出(含预期情景预演)

## 第三步:落盘输出(严格按框架「输出要求」执行)

- 写入 `output/{股票代码}_{YYYYMMDD}_深度分析报告.md`(目录不存在先创建),**不得只在对话中输出文本**。
- 报告视觉结构:报告头 → 决策仪表盘(🟢🟡🔴 状态徽章) → 正文六模块 → 操作计划表 + 情景预演 + 风险提示 → 免责声明。
- 对话回报:文件相对路径 + 一句话结论 + 关键价位(建仓/止损/目标)+ 仓位 + 预期情景。
